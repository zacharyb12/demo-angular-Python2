Guard

il existe plusieurs type de guard sui permettent de configurer la gestion des routes

On va creer un element qui sera appelé sur une route

  • canActivate : permet de savoir si on autorise l'acces a la route

    ng g guard admin-guard

  • canDeactivate : permet de savoir si on autorise la desactivation d'une route

    ng g guard form-guard

  • canActivateChild : permet de savoir si on autorise l'acces a un enfant de la route

    ng g guard child-guard

  • canMatch : permet de verifier avant de charger le composant si l'utilisateur a l'acces

    ng g guard load-guard

Pour utiliser un guard :

le generer avec la commande suivante : ng g guard admin-guard

Definir dans le guard quel sera la condition pour acceder a la router
