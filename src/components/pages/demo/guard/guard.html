Guard

Les guards sont des services spéciaux dans Angular qui permettent de contrôler l'accès aux routes en fonction de certaines conditions.

il existe plusieurs type de guard qui permettent de configurer la gestion des routes

Ils sont utilisés pour protéger les routes sensibles, gérer les autorisations d'accès et améliorer la sécurité globale de l'application.

Chaque interface correspond à un type de guard et définit une méthode qui doit être implémentée pour déterminer si la navigation vers une route doit être autorisée ou non.

  • canActivate : permet de définir si on autorise l'acces à la route

    ng g guard admin-guard

  • canDeactivate : permet de définir si on autorise la désactivation d'une route

    ng g guard form-guard

  • canActivateChild : permet de définir si on autorise l'accès à un enfant de la route

    ng g guard child-guard

  • canMatch : permet de vérifier avant de charger le composant si l'utilisateur a l'accès

    ng g guard load-guard

Pour utiliser un guard :

le generer avec la commande suivante :

ng g guard admin-guard

Definir dans le guard quel sera la condition pour acceder a la route

Dans nos routes ( app.routes.ts ) nous allons ajouter le paramètre sur la route concerné

{ path : 'route' , component : Composant , canActivate : [ 'nom_Du_Guard'] },
